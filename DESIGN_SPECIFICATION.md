# Soseki AI Concierge – デザイン仕様書

**Author**: Manus AI

---

## プロジェクト概要

Soseki AI Conciergeは、文豪・夏目漱石の文体と思想を通じて現代の悩みに新しい視点を与える対話型AIコンシェルジュアプリケーションです。青空文庫に収録された漱石の作品から関連する一節を引用し、ユーザーの質問に応答することで、文学的AI体験を提供します。

本仕様書では、UIモックアップとWebプロトタイプの実装において採用したデザインシステム、カラーパレット、タイポグラフィ、レイアウト原則、コンポーネント設計について詳細に記述します。

---

## デザインコンセプト

### 知的ミニマリズム

明治・大正期の書籍装幀から着想を得た落ち着いた色調と、余白を活かしたミニマルな構成を採用しました。**Calm Interface**の思想に基づき、過度な装飾を避け、情報を整理して提示することで、ユーザーが肩の力を抜いて考えを整理できる体験を目指しています。

### ターゲットユーザー

文学・哲学に興味のある一般層（高校生以上の教養層）、教育関係者（日本文学・国語教育・日本語学習）、文化施設・研究機関（文学館・大学など）を想定しています。

## カラーパレット

Soseki AIのカラーパレットは、明治・大正期の書籍装幀から着想を得た、落ち着いた色調を基調としています。以下の表に、各カラーの用途と具体的な値を示します。

| カラー名 | 用途 | OKLCH値 | 近似HEX値 | 説明 |
|---------|------|---------|-----------|------|
| **Primary** | メインカラー、ヘッダー背景 | `oklch(0.25 0.01 60)` | `#2C2C2C` | チャコールグレー。知性と品格を表現 |
| **Primary Foreground** | Primary上のテキスト | `oklch(0.98 0.005 80)` | `#F5F5F0` | オフホワイト |
| **Background** | ページ背景 | `oklch(0.98 0.005 80)` | `#F5F5F0` | 和紙のような温かみのある背景 |
| **Foreground** | 本文テキスト | `oklch(0.15 0.005 60)` | `#1A1A1A` | ほぼ黒。高い可読性 |
| **Card** | カード背景 | `oklch(1 0 0)` | `#FFFFFF` | 純白。コンテンツを際立たせる |
| **Card Foreground** | カード内テキスト | `oklch(0.15 0.005 60)` | `#1A1A1A` | 本文テキストと同じ |
| **Secondary** | セカンダリ背景 | `oklch(0.95 0.005 80)` | `#EFEFEA` | ライトベージュグレー |
| **Muted** | 控えめな背景 | `oklch(0.92 0.005 80)` | `#E8E8E0` | 補助的な要素に使用 |
| **Muted Foreground** | 補助テキスト | `oklch(0.5 0.01 60)` | `#6B6B6B` | ミディアムグレー |
| **Accent** | アクセントカラー | `oklch(0.75 0.08 80)` | `#C9A961` | 淡い金色。文豪感を演出 |
| **Accent Foreground** | Accent上のテキスト | `oklch(0.15 0.005 60)` | `#1A1A1A` | 高コントラストで可読性確保 |
| **Border** | ボーダー・区切り線 | `oklch(0.9 0.005 80)` | `#E0E0D8` | ベージュグレー。控えめな区切り |
| **Input** | 入力フィールド枠線 | `oklch(0.9 0.005 80)` | `#E0E0D8` | Borderと同じ |
| **Ring** | フォーカスリング | `oklch(0.75 0.08 80)` | `#C9A961` | Accentと同じ金色 |

### カラー使用ガイドライン

カラーパレットは、Tailwind CSS 4のOKLCH形式で定義されています。OKLCH（Lightness, Chroma, Hue）は、知覚的に均一な色空間を提供し、アクセシビリティとデザインの一貫性を向上させます。

すべてのセマンティックカラー（`bg-background`, `text-foreground`など）は、`client/src/index.css`のCSS変数として定義されており、プロジェクト全体で一貫して使用されます。

## タイポグラフィ

日本の文学作品にふさわしい、明朝体を中心とした構成を採用しています。Google Fontsから**Noto Serif JP**と**Noto Sans JP**を読み込み、以下のように使い分けています。

| 要素 | フォント | ウェイト | 用途 |
|------|---------|---------|------|
| **見出し（h1-h6）** | Noto Serif JP | Bold (700) | ページタイトル、セクション見出し。格調高い印象 |
| **本文** | Noto Sans JP | Regular (400) | 通常のテキスト。読みやすさと品格の両立 |
| **引用（.quote）** | Noto Serif JP | Medium (500) | 漱石作品の引用。視覚的に際立たせる |
| **UI要素** | Noto Sans JP | Regular (400) | ボタン、ラベル。モダンな操作性 |

### タイポグラフィ設定

```css
body {
  font-family: 'Noto Sans JP', sans-serif;
}

h1, h2, h3, h4, h5, h6 {
  font-family: 'Noto Serif JP', serif;
  font-weight: 700;
}

.serif {
  font-family: 'Noto Serif JP', serif;
}

.quote {
  font-family: 'Noto Serif JP', serif;
  font-weight: 500;
  border-left: 3px solid oklch(0.75 0.08 80); /* 金色の縦線 */
  padding-left: 1rem;
}
```

引用部分には、左側に金色の縦線（3px）を配置し、文学作品からの引用であることを視覚的に示しています。

## レイアウト原則

### グリッドシステム

12カラムグリッドシステムを採用し、レスポンシブ対応を実現しています。Tailwind CSSの`grid`ユーティリティを活用し、デスクトップでは3カラム、タブレット・モバイルでは1カラムレイアウトに切り替わります。

### 最大コンテンツ幅

読みやすさを重視し、最大コンテンツ幅を**1200px**に設定しています。これにより、長文の読書体験が快適になります。

### 余白スケール

一貫性のある空間設計のため、以下の余白スケールを使用しています。

| サイズ | 値 | 用途 |
|--------|-----|------|
| **Small** | 16px (1rem) | コンポーネント内の小さな余白 |
| **Medium** | 24px (1.5rem) | セクション間の余白 |
| **Large** | 32px (2rem) | ページ全体の余白 |
| **Extra Large** | 48px (3rem) | ヒーローセクションなど大きな余白 |

### ボーダー半径

控えめな角丸（**4px**）を採用し、柔らかさを演出しつつ、過度な装飾を避けています。

```css
--radius: 0.25rem; /* 4px */
```

## コンポーネント設計

### QuoteOfTheDay（今日の一節）

トップページの右側に配置される、日替わりで漱石作品の一節を表示するカードコンポーネントです。

**特徴**
- カード形式（白背景、ボーダー、シャドウ）
- 引用部分は`.quote`クラスで金色の縦線装飾
- 出典情報（作品名・年代）を右下に小さく表示
- 日付ベースでランダムに一節を選択（一日中同じ引用を表示）

**実装ファイル**: `client/src/components/QuoteOfTheDay.tsx`

### AIResponseCard（AI応答カード）

対話ページで使用される、AIの応答を表示するカードコンポーネントです。

**構成要素**
1. **引用部分**: `.quote`クラスで金色の縦線装飾、明朝体
2. **解説部分**: 柔らかい語調で現代的コメント
3. **出典情報**: ボーダー上部に作品名と年代を表示
4. **アクションボタン**: 「もう少し聞く」「別の一節を」の2つのボタン

**実装ファイル**: `client/src/components/AIResponseCard.tsx`

### RelatedWorks（関連作品リスト）

対話ページのサイドバーに表示される、漱石の代表作品リストです。

**特徴**
- カード形式でスティッキー配置（`sticky top-4`）
- 各作品に金色の左ボーダー（2px）
- 作品名（明朝体）、年代、短い説明文を表示
- デスクトップのみ表示、モバイルでは非表示

**実装ファイル**: `client/src/components/RelatedWorks.tsx`

## ページ構成

### トップページ（Home）

ユーザーが最初に訪れるページで、質問入力とQuote of the Dayを表示します。

**レイアウト**
- ヒーローセクション: 中央配置のタイトル＋サブコピー
- 3カラムグリッド（デスクトップ）: 入力エリア（2カラム）+ Quote of the Day（1カラム）
- 1カラムレイアウト（タブレット・モバイル）

**主要要素**
- タイトル: `Soseki AI Concierge`（大きな見出し、明朝体）
- サブコピー: `漱石があなたに語りかける日常の文話（bunwa）`（明朝体、控えめな色）
- 入力ボックス: 大きめのテキストエリア（120px高）
- 送信ボタン: Primaryカラー、矢印アイコン付き
- 紹介文: アプリケーションの説明（本文フォント）
- フッター: 教育・文化用途の注記、引用元情報

**実装ファイル**: `client/src/pages/Home.tsx`

### 対話ページ（Chat）

ユーザーとAIの対話が展開されるページです。

**レイアウト**
- ヘッダー: Primaryカラー背景、タイトル、戻るボタン、設定ボタン
- 3カラムグリッド（デスクトップ）: 対話エリア（2カラム）+ サイドバー（1カラム）
- 1カラムレイアウト（タブレット・モバイル）: サイドバー非表示

**主要要素**
- メッセージ表示: ユーザーメッセージ（右寄せ、Secondaryカラー背景）、AIメッセージ（左寄せ、AIResponseCard）
- 入力エリア: スティッキー配置（`sticky bottom-4`）、テキストエリア＋送信ボタン
- サイドバー: RelatedWorksコンポーネント（デスクトップのみ）

**アニメーション**
- メッセージ追加時に`animate-fade-in`クラスでフェードイン効果
- スムーズなスクロール（`scroll-behavior: smooth`）

**実装ファイル**: `client/src/pages/Chat.tsx`

### 404ページ（NotFound）

存在しないページにアクセスした際に表示されるエラーページです。

**特徴**
- 中央配置のシンプルなレイアウト
- 大きな「404」見出し
- 日本語の説明文
- ホームに戻るボタン（Primaryカラー）

**実装ファイル**: `client/src/pages/NotFound.tsx`

## レスポンシブ対応

### ブレークポイント

Tailwind CSSのデフォルトブレークポイントを使用しています。

| ブレークポイント | 幅 | デバイス | レイアウト |
|-----------------|-----|---------|-----------|
| **Mobile** | < 640px | スマートフォン | 1カラム、サイドバー非表示 |
| **Tablet** | 640px - 1023px | タブレット | 1カラム、サイドバー非表示 |
| **Desktop** | ≥ 1024px | デスクトップ | 3カラム、サイドバー表示 |

### レスポンシブ戦略

**デスクトップ（≥ 1024px）**
- トップページ: 入力エリア（2カラム）+ Quote of the Day（1カラム）
- 対話ページ: 対話エリア（2カラム）+ サイドバー（1カラム）

**タブレット・モバイル（< 1024px）**
- すべてのページで1カラムレイアウト
- サイドバーは非表示（`hidden lg:block`）
- 入力ボックスとボタンのサイズを調整
- タッチ操作に最適化されたボタンサイズ

## アニメーションとトランジション

### フェードインアニメーション

メッセージ追加時に使用するフェードインアニメーションを定義しています。

```css
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(10px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.animate-fade-in {
  animation: fadeIn 0.5s ease-out;
}
```

### スムーズトランジション

ボタンやリンクのホバー効果に、0.3秒のスムーズなトランジションを適用しています。

```css
.transition-smooth {
  transition: all 0.3s ease-in-out;
}
```

## アクセシビリティ

### カラーコントラスト

すべてのテキストとボタンは、WCAG 2.1 AA基準（コントラスト比4.5:1以上）を満たすように設計されています。特に、Primaryカラー（チャコールグレー）とPrimary Foreground（オフホワイト）の組み合わせは、十分なコントラストを確保しています。

### フォーカスリング

キーボード操作時に、フォーカスリング（金色、`--ring`）が表示され、現在のフォーカス位置が明確になります。

### セマンティックHTML

見出しタグ（`h1`, `h2`, `h3`など）を適切に使用し、スクリーンリーダーでの読み上げを考慮しています。

### キーボード操作

すべてのインタラクティブ要素（ボタン、リンク、入力フィールド）はキーボードで操作可能です。特に、テキストエリアでは`Enter`キーで送信、`Shift+Enter`で改行ができるように実装しています。

## 今後の拡張可能性

### バックエンド統合

現在はフロントエンドのみのプロトタイプですが、以下の機能を追加することで、実用的なアプリケーションに発展できます。

**推奨技術スタック**
- **RAG（Retrieval-Augmented Generation）**: Faiss または Milvus を使用した引用検索
- **埋め込みモデル**: bge-m3（日本語＋英語対応）
- **生成モデル**: Qwen 2.5 / Mistral 7B Instruct
- **オーケストレーション**: LangChain / LlamaIndex

### 追加機能

**トーン設定**
- Formal（格式ばった語調）/ Gentle（柔らかい語調）の切り替え

**多言語対応**
- 英語併記モード（高等レベル語彙に英語を併記）

**音声読み上げ**
- Text-to-Speech機能で引用を音声で聞く

**お気に入り機能**
- 気に入った引用を保存し、後で見返す

**テーマ別検索**
- 孤独・友情・仕事・芸術・恋愛などのテーマで引用を検索

## まとめ

Soseki AI Conciergeは、夏目漱石の文学世界を現代的なUIで体験できる対話型アプリケーションです。本仕様書で定義したデザインシステムに従うことで、一貫性のある美しいUIを維持しながら、機能を拡張していくことができます。

**デザインの核心**は、**知的ミニマリズム**と**Calm Interface**の思想です。過度な装飾を避け、余白を活かし、情報を整理して提示することで、ユーザーが文学的な対話に集中できる環境を提供しています。

今後の開発において、本仕様書を参照し、デザインの一貫性を保ちながら、新しい機能を追加していくことを推奨します。
